# Examples

These artifacts were generated by running:

```bash
python3 scripts/generate_examples.py
```

The generated context mirrors the seeded DataHub demo lineage:

```text
raw_transactions -> user_risk_features -> credit_risk_v3 -> credit_risk_prod -> risk_ops_dashboard
```

Intentional risks represented in the output:

- schema drift: `raw_transactions.customer_id` changed from `int` to `string`
- PII exposure: `raw_transactions.email` flows into model lineage
- stale data: `raw_transactions` exceeded its 24-hour refresh cadence
- missing owner: `user_risk_features` has no owner
- feature leakage: `chargeback_resolved_at` is post-outcome evidence

Files:

- `sample_report.json`: machine-readable report
- `sample_report.html`: static visual lineage report
- `sample_report.md`: PR-comment-ready summary
- `sample_mcp_dryrun.json`: DataHub MCP payloads that would be emitted in dry-run mode
- `sample_writeback_audit.jsonl`: JSONL audit trail for dry-run write-back decisions

Live DataHub examples can be regenerated with `make demo` on a machine with Docker.
